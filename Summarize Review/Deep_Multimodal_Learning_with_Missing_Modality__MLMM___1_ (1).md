1. Source and Scope: This document summarizes Wu, Wang, Chen, and Carneiro's (2026) survey "Deep Multimodal Learning with Missing Modality" (arXiv:2409.07825v4), the first comprehensive review of MLMM, compiling 354 papers from 2012–2025.
2. Research Motivation: Real-world multimodal systems frequently encounter missing modalities due to sensor malfunctions, hardware limitations, privacy restrictions, environmental interference, and transmission failures, motivating robust handling strategies.
3. Problem Formulation: A missing-modality sample lacks one or more of N expected modalities (e.g., having only Mod 1 and Mod 3 when Mod 2 is expected), as opposed to full-modality samples containing all modalities.
4. Central MLMM Goal: To robustly exploit available modalities while maintaining performance comparable to systems trained with complete modality information, distinguishing MLMM from conventional multimodal learning which assumes full modality availability.
5. Historical Precedents: Early affective computing research (facial/audio occlusion and noise), medical AI (privacy-restricted modalities), and NASA's Mars Ingenuity helicopter (sensor failure) all motivated missing-modality robustness research.
6. Survey Methodology: Papers were sourced from Google Scholar and top venues (AAAI, NeurIPS, ICML, CVPR, MICCAI, etc.) using keywords like "incomplete," "missing," and "partial" combined with multimodal/deep learning terms.
7. Two-Aspect Taxonomy: MLMM methods are categorized by Data Processing aspect (Modality Imputation, Representation-Focused Models) and Strategy Design aspect (Architecture-Focused Models, Model Combinations).
8. Modality Imputation Definition: Operates at the data level, filling missing modalities via composition (zero/random values, retrieved similar instances) or generation (using generative models like AEs, GANs, diffusion models).
9. Zero/Random Composition Method: Replaces missing modality data with zeros or random values; commonly used as a simple baseline that prevents over-reliance on dominant modalities but does not address true information recovery.
10. Retrieval-Based Composition: Fills missing data by copying or averaging same-category samples (e.g., via KNN); performs better than zero/random methods and can handle test-time missingness, but suffers from high computational cost, memory overhead, and overfitting risk.
11. Individual Modality Generation Methods: Train a dedicated generative model per modality (e.g., AE-based PET-from-MRI generation, GAN-based SensorGAN, diffusion-based IMDer) to synthesize missing data from available modalities.
12. Unified Generation Methods: Train a single generator (e.g., Cascade AE) to produce all modalities simultaneously, though this increases storage/computation and complicates training as modality count grows.
13. Representation-Focused Models Definition: Address missing modalities at the feature/representation level rather than raw data level, via coordinated representations, representation composition, or representation generation.
14. Coordinated-Representation Methods: Impose constraints (Tensor Rank Regularization, Deep CCA, Hilbert-Schmidt Independence Criterion) to align representations across modalities; effective mainly with two or three modalities, degrading as modality count increases.
15. Representation Composition (Retrieval-Based): Recovers missing representations by retrieving and averaging top-K similar features from a feature pool using cosine similarity.
16. Representation Composition (Arithmetic): Flexibly fuses any number of modality representations via parameter-free operations (pooling, addition, sign-max merging, token merging like TMFormer), trading inter-modality relationship capture for flexibility.
17. Representation Generation (Indirect-to-Task): Uses an auxiliary reconstruction task (encoder-decoder) during training that is discarded at inference; sub-divided into Before-Fusion (e.g., MGP-VAE, PFNet) and After-Fusion (e.g., ActionMAE, M3AE) variants; requires missing modality data during training.
18. Representation Generation (Direct-to-Task): Trains a small generator (e.g., Hallucination Network, SMIL with Bayesian meta-regularization, RMM for AVQA) to map available representations directly to missing ones without raw data reconstruction; generally underperforms indirect-to-task methods.
19. Architecture-Focused Models Overview: Adjust model architecture flexibly to handle varying modality availability via four techniques—attention, distillation, graph learning, and MLLMs.
20. Attention Fusion Methods: Intra-modality attention computes attention per modality independently before sharing fusion tokens (e.g., BEV-Evolving Decoder); inter-modality attention uses masked attention to make the model ignore missing modality slots (e.g., DrFuse).
21. Transformer-Based Methods: Joint Representation Learning (JRL) fully trains parameters to learn from any number of modality tokens (e.g., Masked Multimodal Transformers); Parameter Efficient Learning (PEL) uses prompt tuning or adapter tuning, fine-tuning as little as 1% of parameters but with lower performance than JRL.
22. Distillation-Based Methods: Knowledge distillation transfers information from a full-modality teacher to a missing-modality student via response distillation (logits) or intermediate distillation (features); most assume complete training data, limiting use for incomplete training datasets.
23. Process-Based Distillation: Mean Teacher Distillation uses an exponential moving average of student parameters as a stable teacher; self-distillation (e.g., ShaSpec) lets models learn from their own branch representations.
24. Graph Learning-Based Methods: Graph fusion methods (e.g., HGMF with hyper edges) integrate multimodal data via graph structures; GNN methods either process available modalities individually or first complete the graph (e.g., FeatProp) before GNN processing; limited by scalability for large datasets.
25. Multimodal Large Language Models (MLLMs): Leverage transformer flexibility and long-context handling to naturally process any number of modalities, exemplified by BLIP-2, LLaVA, AnyGPT, NExT-GPT, and CoDi; limited by positional encoding inconsistency, training difficulty, and lack of dedicated missing-modality benchmarks.
26. Model Combinations Overview: Use multiple models or specialized training strategies, divided into Ensemble, Dedicated Training, and Discrete Scheduler methods.
27. Ensemble Methods: Combine multimodal model outputs (e.g., weighted judgment reconstruction) or unimodal model outputs (e.g., probabilistic ensemble, Uni-Modal Ensemble with Missing Modality Adaptation) to handle missing inputs flexibly.
28. Dedicated Training Methods: Train specialized models for each modality combination (up to seven for three modalities), exemplified by KDNet (distillation-based) and ACNet (adversarial co-training).
29. Discrete Scheduler Methods: Use an LLM as a controller to orchestrate task-specific modules (Visual ChatGPT, HuggingGPT, ViperGPT), enabling flexible handling of varying modality counts but requiring fast, accurate LLM instruction interpretation.
30. Recovery vs. Non-Recovery Taxonomy: An alternative classification splits methods by whether they recover missing information (75.5% of works) or perform inference without recovery (24.5%), further subdivided into early/intermediate/late/hybrid integration stages.
31. Intermediate-Stage Dominance: Intermediate-stage recovery accounts for 45.8% of recovery methods—the largest share—because feature-level recovery avoids raw-data noise while retaining rich modality-specific information.
32. Technical Evolution Phases: The field progressed through four phases—Early Heuristic Era (pre-2017, data-driven heuristics/ensembles), Representation Learning Era (2017+, coordinated/generative representations), Architecture-Focused Era (post-2018, attention/distillation/graph methods), and Foundation Model Era (most recent, transformer-based universal processors).
33. Cross-Domain Architecture Transfer: Modern architectures like attention mechanisms transfer across unrelated application domains (e.g., GAN recovery used in both cancer prognosis and person re-identification), showing weak linkage between application and architecture choice.
34. Application Domain Coverage: MLMM methods are applied across affective computing, medical AI, information retrieval, remote sensing, pervasive computing, robotic vision, and natural science applications.
35. Affective Computing Datasets: Span video/audio/text/bio-sensor combinations for emotion classification, including CMU-MOSI, CMU-MOSEI, IEMOCAP, and comic-based datasets like Manga109.
36. Medical Applications: Cover neuroimaging, cardiovascular, cancer detection, ophthalmology, and computational pathology, with datasets including BRATS, ADNI, MIMIC-CXR, and MIMIC-IV; missing modalities arise from privacy concerns and invasive-procedure data gaps.
37. Remote Sensing Challenge: Optical modalities may be unavailable due to cloud cover or sensor damage, addressed using datasets like WHU-OPT-SAR and DFC2020.
38. Pervasive Computing Domains: Covers human activity recognition (motion sensors), physiological monitoring (biosignals like ECG/EDA), and cognitive state inference (EEG), all prone to structural missingness from sensor dropout and irregular sampling.
39. Robotic Vision Tasks: Includes multimodal segmentation, detection, activity recognition, person re-identification, and face anti-spoofing using RGB+X (depth, LiDAR, radar, infrared) sensor combinations.
40. Dataset Missingness Statistics: In severe cases up to 90% of samples may have missing modalities; 36.4% of surveyed works train on incomplete datasets while 63.6% train on complete data and test only with missing modalities.
41. Missing Rate Severity Findings: Across surveyed works, mild missingness (<30%) appears in 72.6% of studies, moderate (30–70%) in 82.2%, and severe (>70%) in 69.9%, with single studies often validating across multiple levels.
42. Open Issue—Accurate Generation: Generated/recovered missing modality content often contains artifacts or hallucinations; future work should better exploit modality complementarity and redundancy to generate unbiased, task-relevant data.
43. Open Issue—Recovery vs. Non-Recovery Decision: No principled framework currently exists for deciding when to recover missing modalities versus relying solely on available ones, especially given risks of recovered information being ignored or dominating incorrectly.
44. Open Issue—Benchmarking: Inconsistent missing-modality settings across studies hinder fair comparison; the field needs a unified benchmark (similar to MultiBench) covering diverse datasets, missing rates, and patterns.
45. Open Issue—Efficiency: Many MLMM methods (especially model-combination approaches) are computationally heavy, posing challenges for deployment on resource-constrained devices like space or disaster-response robots.
46. Open Issue—Streaming/Temporal Data: Limited research addresses asynchronous missing modalities in streaming/temporal data, where misalignment compounds recovery and fusion challenges.
47. Open Issue—Reinforcement Learning: Only a few works (e.g., Vasco et al. 2021) address missing modalities in multimodal reinforcement learning contexts like robotic grasping and driving.
48. Open Issue—Natural Science Applications: Drug prediction and materials science face missing modalities due to data access restrictions and acquisition costs, with limited dedicated MLMM research so far.
49. Open Issue—Real-World IoT/Wearables: Structural, persistent missingness from heterogeneous devices, battery limits, and intermittent connectivity in IoT/wearable settings requires general frameworks for long-term partial observability, which current methods lack.
50. System Architecture Pipeline: The generalized MLMM architecture pipeline includes (1) per-modality encoding only for available modalities, (2) missing-modality mask generation, (3) representation alignment, (4) gated fusion without forced zero-fill, and (5) downstream task prediction.
51. No Forced Zero-Fill Principle: Replacing missing modalities with zero vectors injects zero-information tokens and is discouraged; preferred approaches skip absent modalities entirely or explicitly generate their representations before fusion.
52. Dynamic Input Dimensionality Principle: Robust fusion mechanisms must handle any subset of available modalities via pooling operations or attention mechanisms that naturally accommodate variable-length inputs.
53. Modality Importance Weighting Principle: Gating weights should be learned or dynamically inferred via attention to reflect each available modality's informational contribution to the downstream task.
54. Representation-Level vs. Data-Level Recovery Principle: Representation-level recovery is generally preferred over data-level recovery because features carry more generalized, less noisy information.
55. Training vs. Inference Distinction Principle: Methods must separately address training-time missingness (requiring augmentation, reconstruction, or teacher distillation) versus inference-time missingness (requiring runtime adaptation via masks, generators, or attention routing).
56. Overall Conclusion: The field has evolved from simple data-level heuristics through representation learning to sophisticated architecture-level solutions, with intermediate-stage recovery dominating due to its noise-avoidance and information-richness tradeoff.
57. Future Trajectory: Missing-modality robustness is expected to become a built-in capability of general-purpose multimodal foundation models rather than a specialized add-on, as transformer-based and MLLM architectures increasingly process variable modality sets natively.
